## RNA Models

!!! warning

    The BAM format does not support `U` bases.

    Therefore, when Dorado is performing RNA basecalling, the resulting output files will include `T` instead of `U`.
    This is consistent across output file types.

    The same applies to parsing inputs.
    Any input HTS file (e.g. FASTQ generated by `guppy`/`basecall_server`) with `U` bases is not handled by `dorado`.
